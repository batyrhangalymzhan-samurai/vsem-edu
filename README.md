# Сайт «ВСЕМ ЕДУ»

Ссылка на онлайн-меню: https://vsem-edu.vercel.app

Хостинг-платформа: Vercel

Статический сайт (HTML/CSS/JS без сборки) со встроенной **CMS-админкой** для удобного
редактирования текстов и фотографий без правки кода.

## Структура проекта

```
index.html          — главная страница (весь контент подгружается из content.json)
style.css           — стили сайта
script.js           — рендер сайта из content.json + интерактивность (вкладки, модалка фото)
content.json        — ВЕСЬ контент сайта: тексты, телефон, часы, меню, отзывы, фото
images/             — фотографии (логотипы, блюда, отзывы)

admin/               — CMS-админка
  index.html          — интерфейс админки
  admin.css           — стили админки
  admin.js            — логика админки (загрузка фото, сохранение контента)

api/                 — серверная часть админки (Vercel Serverless Functions)
  login.js            — проверка пароля, выдача сессионного токена
  content.js           — чтение/сохранение content.json
  upload.js            — загрузка фотографий
  _github.js            — работа с GitHub Contents API
  _auth.js               — подпись и проверка сессионных токенов
```

## Как это работает

Админка не хранит собственную базу данных. При сохранении она напрямую обновляет
файл `content.json` (и загружает фото в папку `images/`) **в GitHub-репозитории**
через GitHub API. Vercel следит за репозиторием и автоматически пересобирает
и обновляет сайт после каждого такого изменения — обычно это занимает 30–60 секунд.

```
Вы редактируете сайт в /admin
        │
        ▼
Админка сохраняет content.json и фото в GitHub (через API)
        │
        ▼
Vercel видит изменение в репозитории и автоматически деплоит сайт
        │
        ▼
Изменения появляются на сайте
```

## Настройка (один раз)

Чтобы админка заработала, нужно один раз задать несколько переменных окружения
в настройках проекта на Vercel: **Project → Settings → Environment Variables**.

| Переменная       | Что это                                                                 |
|------------------|--------------------------------------------------------------------------|
| `GITHUB_OWNER`   | Логин на GitHub, где лежит репозиторий сайта                             |
| `GITHUB_REPO`    | Название репозитория                                                     |
| `GITHUB_BRANCH`  | Ветка, из которой деплоится сайт (обычно `main`)                         |
| `GITHUB_TOKEN`   | Personal Access Token от GitHub с доступом на запись в репозиторий       |
| `ADMIN_PASSWORD` | Пароль для входа в `/admin`                                              |
| `SESSION_SECRET` | Любая случайная строка (для подписи сессии администратора)               |

### Как получить `GITHUB_TOKEN`

1. Зайдите на GitHub → **Settings → Developer settings → Personal access tokens → Fine-grained tokens**.
2. Создайте новый токен, выберите нужный репозиторий.
3. В разделе **Repository permissions** дайте право **Contents: Read and write**.
4. Скопируйте токен и вставьте его в переменную `GITHUB_TOKEN` на Vercel.

После добавления переменных сделайте redeploy проекта на Vercel, чтобы они применились.

Пример значений — в файле `.env.example`.

## Как пользоваться админкой

1. Откройте `https://ваш-сайт.vercel.app/admin`.
2. Введите пароль (`ADMIN_PASSWORD`).
3. Во вкладке **«Общее»** — телефон, часы работы, сумма доставки, WhatsApp,
   Instagram, заголовок главного экрана, логотипы, футер.
4. Во вкладке **«Меню»** — категории блюд: можно добавлять/удалять категории,
   менять порядок (стрелки ↑/↓), добавлять/удалять и заменять фото блюд.
5. Во вкладке **«Отзывы»** — фотографии отзывов из Instagram: добавление,
   удаление, замена.
6. Для загрузки фото просто нажмите на изображение (или «Загрузить») и выберите
   файл — оно автоматически сжимается и сохраняется.
7. После внесения изменений обязательно нажмите **«Сохранить изменения»**
   в соответствующей вкладке. Изменения появятся на сайте через 30–60 секунд.

## Локальная разработка

Сайт можно открыть локально просто через `index.html`, но для работы `/admin`
и `/api` нужен [Vercel CLI](https://vercel.com/docs/cli):

```bash
npm i -g vercel
vercel dev
```

Перед запуском скопируйте `.env.example` в `.env` и заполните значениями.
